## 항목별 세부 평가

### ✅ 잘 된 점

**1. CSS 변수 아키텍처**
- `palettes.css` 단일 소스 → `style.css` alias → 브라우저 카스케이드 구조가 명확
- 5팔레트 × 라이트/다크 모드 = 10가지 테마가 단 2개 파일로 관리됨
- `--p1-on-surface` 컨텍스트 토큰으로 WCAG AA 대비비 자동 전환

**2. Open Graph 구현**
- 필수 6개 태그 (`og:url`, `og:title`, `og:description`, `og:type`, `og:image`, `og:site_name`) 완비
- `og:image:width/height` 명시로 크롤러 렌더링 최적화
- 카카오톡 실제 공유 테스트 통과

**3. 파비콘 시스템**
- `favicon.svg` (모던 브라우저), `favicon.png` (폴백), `apple-touch-icon.png` (iOS) 3단계 제공
- SVG 파비콘으로 Retina 디스플레이에서 선명하게 표시

**4. OG 이미지 디자인 일관성**
- 사이트 히어로 섹션 다크 네이비 팔레트와 동일한 색상 사용
- 뉴런 네트워크 일러스트가 뇨로사이언스 연구자 정체성과 일치
- 캔버스 기반 `generate-og.html` 제공으로 향후 텍스트 수정 용이

### ⚠️ 개선 필요 사항

**1. Imaging & Histology 카드 모바일 레이아웃 (진행 중)**
- 문제: 4개 그리드 아이템이 3칸 그리드에 잘못 배치되어 p 텍스트 폭 축소
- 대응: HTML wrapper 추가로 3개 아이템 구조로 변경 (오늘 최종 커밋)
- 상태: 배포 완료, 모바일 재확인 필요

**2. OG 이미지 한국어 폰트**
- `generate-og.html`의 Canvas는 시스템 폰트(`Malgun Gothic`)에 의존
- macOS/Linux 환경에서 생성 시 한국어 폰트가 다르게 렌더링될 수 있음
- 개선안: 서버사이드 이미지 생성(Satori, Puppeteer) 또는 Google Fonts 임베드 고려

**3. 팔레트 전환 애니메이션**
- 현재 팔레트 변경 시 CSS 전환 효과 없이 즉시 색상이 바뀜
- `transition: background-color 0.3s, border-color 0.3s` 추가 고려

**4. 시멘틱 HTML 개선 여지**
- `<div class="sk-highlight">` → `<mark>` 또는 `<strong>` 태그 검토
- 스킬 목록을 `<ul>/<li>` 구조로 변경 시 스크린 리더 접근성 향상
